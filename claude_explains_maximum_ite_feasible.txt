Here's the full analysis of why ITE correlations are modest even with oracle confounders.                                                                          
                                                                                                                                                                     
  Why Oracle Explicit Confounder ITE Correlations Are Modest                                                                                                         
                                                                                                                                                                     
  The results show r ≈ 0.49 (1K) and r ≈ 0.68 (50K) even with true confounder values. This is explained by four compounding factors, with the first being dominant:  
                                                                                                                                                                     
  1. Binary Outcome Noise (Primary Factor)
                                                                                                                                                                     
  This is the single biggest issue. The causal forest observes binary Y ∈ {0, 1}, not the underlying probabilities. The true ITE is a difference in probabilities (τ
  = P(Y=1|T=1,X) - P(Y=1|T=0,X)), but each observation is an extremely noisy Bernoulli draw from that probability.
  ┌───────────────────────┬───────┐
  │       Quantity        │ Value │
  ├───────────────────────┼───────┤
  │ ITE signal (std)      │ 0.237 │
  ├───────────────────────┼───────┤
  │ Bernoulli noise (std) │ 0.557 │
  ├───────────────────────┼───────┤
  │ SNR                   │ 0.43  │
  └───────────────────────┴───────┘
  The noise is 2.3x larger than the signal. When I gave the causal forest the true probabilities instead of binary outcomes, correlation jumped from r = 0.55 → 0.75
  — the binary discretization alone costs ~0.20 in correlation.

Ken note 2-7-26: this is probably a realistic feature given true stochasticity in clinical outcomes especially.

  2. Small Sample Size (N = 1000)

  CausalForestDML with honest estimation splits data in half — one half builds tree structure, the other estimates effects. With 5-fold CV and honest splitting, each
   leaf estimate draws from an effective N ≈ 400. A leaf with ~10 samples (5 treated, 5 control) has per-leaf effect estimate std ≈ 0.45, which is nearly 2x the ITE
  std itself. The improvement from 1K → 50K (r: 0.49 → 0.68) confirms this.

  3. Nonlinear ITE Function

  The true ITE = σ(logit₀ + Δ) − σ(logit₀) is inherently nonlinear in the confounders due to the sigmoid. Even an oracle supervised regression (Ridge) from true
  confounders directly to true ITE achieves only r = 0.90, not 1.0. The sigmoid compresses effects for patients with extreme baseline probabilities:

  | y0_prob range | Mean |ITE| | N |
  |---------------|-----------|-----|
  | [0.0, 0.1) | 0.147 | 93 |
  | [0.3, 0.7) | 0.212 | 358 |
  | [0.9, 1.0) | 0.077 | 115 |

  4. Fundamental Counterfactual Problem

  Each patient is observed under only one treatment condition. The forest must infer τ(X) by comparing different patients' outcomes, relying on the conditional
  independence assumption. This is fundamentally harder than supervised prediction — there's no single patient for whom you observe both Y(0) and Y(1).

  The core takeaway: r ≈ 0.49–0.55 on 1K binary data is not a failure — it's roughly what's achievable given the SNR. The confounders contain the right information
  (proven by r = 0.90 in supervised regression), but the causal estimation problem with binary outcomes and N = 1000 is fundamentally noisy.
